#### 1. 语音识别功能（Whisper） ⏳ 预计1周

**目标：** 让"今天吃了啥"功能真正可用

**待实现：**
- [ ] **1.1 下载Whisper模型文件**
  - 下载 `ggml-base-q8_0.bin` 或更小版本（如 `ggml-tiny.bin`）
  - 放置到 `app/src/main/assets/whisper/` 目录
  - 文件大小：约 75MB（base）或 75MB（tiny）

- [ ] **1.2 启用NDK编译**
  - 取消注释 `app/build.gradle.kts` 中的 `externalNativeBuild` 配置（第27-40行、104-112行）
  - 确保NDK版本兼容（当前配置为25.1.8937393）

- [ ] **1.3 完善Whisper C++实现**
  - 实现 `whisper/whisper_processor.cpp` 中的 `init()` 方法
  - 实现 `whisper/whisper_processor.cpp` 中的 `transcribe()` 方法
  - 集成 whisper.cpp 库（需要下载或编译）

- [ ] **1.4 创建Kotlin封装类**
  - 创建 `com.eldercare.ai.whisper.WhisperProcessor.kt`
  - 封装JNI调用：`nativeInit()` 和 `nativeTranscribe()`
  - 在 `MainActivity` 中添加 `System.loadLibrary("eldercare-ai")`

- [ ] **1.5 实现音频录制**
  - 使用 `AudioRecord` 录制PCM音频
  - 采样率：16000Hz，单声道，16位
  - 实时录制并转换为float数组

- [ ] **1.6 集成到VoiceDiaryScreen**
  - 替换模拟数据为真实Whisper识别
  - 添加录音状态显示（波形动画）
  - 处理识别结果并保存到数据库

- [ ] **1.7 情感分析（可选）**
  - 从识别文本中提取情感关键词
  - 简单规则匹配：开心/满意/担心等

**技术难点：**
- Whisper模型文件较大，需要优化加载策略
- 音频格式转换（PCM → float array）
- 实时录音和识别性能优化

#### 2. YOLO物体检测功能 ⏳ 预计1周

**目标：** 让"拍冰箱"功能真正可用

**待实现：**
- [ ] **2.1 下载YOLO模型文件**
  - 下载 YOLOv8n（nano版本，最小最快）
  - 需要 `.param` 和 `.bin` 文件
  - 放置到 `app/src/main/assets/yolo/` 目录
  - 文件大小：约 6MB（.param + .bin）

- [ ] **2.2 集成NCNN库**
  - 下载NCNN Android库（.so文件）
  - 放置到 `app/src/main/jniLibs/` 对应架构目录
  - 或使用CMake编译NCNN

- [ ] **2.3 完善YOLO C++实现**
  - 实现 `yolo/yolo_detector.cpp` 中的 `init()` 方法
  - 实现 `yolo/yolo_detector.cpp` 中的 `detect()` 方法
  - 图像预处理（resize、归一化）
  - 后处理（NMS、置信度过滤）

- [ ] **2.4 创建Kotlin封装类**
  - 创建 `com.eldercare.ai.yolo.YoloDetector.kt`
  - 封装JNI调用：`nativeInit()` 和 `nativeDetectObjects()`
  - Bitmap转检测输入格式

- [ ] **2.5 食材识别映射**
  - 创建食材类别映射表（COCO类别 → 中文食材名）
  - 常见食材：蔬菜、水果、肉类、蛋奶、调料等
  - 支持自定义食材类别扩展

- [ ] **2.6 保质期计算逻辑**
  - 根据食材类型设置默认保质期
  - 蔬菜：3-7天，肉类：1-3天，蛋类：7-14天等
  - 根据拍摄时间戳计算过期日期

- [ ] **2.7 过期提醒算法**
  - 计算距离过期天数
  - 分类：新鲜（>3天）、即将过期（1-3天）、已过期（<0天）
  - 生成提醒消息

- [ ] **2.8 集成到FridgeScreen**
  - 替换模拟数据为真实YOLO检测
  - 显示检测框和置信度
  - 保存检测结果到数据库
  - 显示过期提醒列表

**技术难点：**
- NCNN库集成和编译
- 检测框绘制和UI展示
- 多物体检测结果合并去重

  - 匹配可制作的菜品/食谱

  - 显示推荐菜品列表

   - 每个功能完成后需要充分测试
