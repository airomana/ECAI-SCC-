银发智慧膳食管家 项目计划书
ElderCare AI - 让科技消除代沟，用温度守护健康

一、项目概述
1.1 项目背景与社会痛点
真实的数字鸿沟
根据第七次全国人口普查数据，中国60岁及以上人口已达2.64亿，占总人口的18.7%。在数字化浪潮中，老年群体面临严峻的"数字鸿沟"困境：
痛点1 - 看不懂：餐厅菜单上的"舒芙蕾""分子料理""轻食沙拉"等新式菜名，让老年人无所适从；传统的热量表APP使用复杂的营养学术语，老人难以理解。
痛点2 - 记不住：慢病老人（高血压、糖尿病患者占老年人50%以上）需要严格的饮食控制，但容易忘记哪些食物不能吃、今天吃了什么。
痛点3 - 没人陪：子女工作繁忙，独居老人占比超30%，缺乏饮食指导和情感陪伴，导致营养不良或暴饮暴食。
1.2 项目目标
打造一款专为银发族设计的智能饮食管理助手，通过"视觉+语音"的零门槛交互，解决老年人在餐饮场景中的三大核心需求：
1.看得懂 - 拍照识菜，大白话翻译
2.吃得对 - 智能匹配健康档案，主动预警
3.有人陪 - 语音交互陪伴，子女远程守护
1.3 核心创新点
创新点1：知识图谱驱动的"大白话"翻译引擎
突破传统OCR的局限：不仅识别文字，更要"翻译"成老人能理解的语言。
技术实现：
菜单文字 "地三鲜" 
    ↓ OCR识别
菜品知识库查询 → 成分：茄子、土豆、青椒 + 烹饪方式：油炸
    ↓ LLM大白话生成
语音输出："张大爷，这个地三鲜就是茄子土豆炸的，油比较多，您有高血压要少吃哦"
创新价值：首次将专业营养知识"适老化"转译，让AI成为老人身边的"营养师"。
创新点2：基于语音日记的情感陪伴系统
核心理念：不做冷冰冰的工具，做有温度的陪伴者。
实现方式：
老人每日语音记录："今天吃了清蒸鱼，还不错"
系统分析语音中的情感关键词（"挺好""唉""又是一个人"）
生成"饮食与情绪周报"发送给子女，及时发现独居老人的孤独信号
创新价值：将饮食管理与心理关怀结合，关注老人的身心健康。
创新点3：端云协同的隐私保护架构
痛点：老年人对隐私泄露高度敏感，不信任"把照片传到云端"。
解决方案：
端侧处理：OCR文字提取、菜品基础识别在OPPO手机本地完成
云侧增强：仅在复杂场景（模糊图片、生僻菜名）时调用云端大模型
数据加密：健康档案仅在本地存储，采用TEE可信执行环境加密
创新价值：既保证AI能力，又让老人用得放心。

2.2 核心功能模块
功能模块1：智能菜单识别与大白话翻译
使用场景：
李大爷在社区食堂，今日菜单有"雪菜黄鱼""糖醋里脊""清炒时蔬"。他不确定哪个适合自己的糖尿病饮食。
操作流程：
1.点击APP首页超大红色按钮"拍菜单"
2.对准菜单拍照（支持手抖校正）
3.等待2秒，手机语音播报每道菜的"大白话"说明
4.点击感兴趣的菜，听详细分析
技术实现路径：
Step 1: 图像预处理
- 输入：老人拍摄的菜单照片（可能模糊、反光、歪斜）
- 处理：去噪、透视校正、光照均衡
- 工具：OpenCV

Step 2: 文字识别（端侧）
- 引擎：PaddleOCR-mobile（轻量化版本，15MB）
- 输出：菜名列表 ["雪菜黄鱼", "糖醋里脊", "清炒时蔬"]
- 优化：针对中文菜名常用字体训练，识别率>92%

Step 3: 菜品知识匹配（端侧/云侧混合）
- 本地优先：查询内置500道常见菜知识库
  - "糖醋里脊" → 成分：猪肉+糖+醋 → 标签：高糖、高油
- 云端兜底：遇到生僻菜名调用Qwen-VL视觉大模型
  - 上传图片+菜名，返回成分分析

Step 4: 健康规则匹配（端侧）
- 读取用户健康档案：糖尿病、高血压
- 规则引擎判断：
  if 菜品标签含"高糖" and 用户有"糖尿病":
      风险等级 = "红色警告"

Step 5: 大白话生成（LLM）
- Prompt工程：
  "将以下营养分析转换为适合70岁老人理解的口语化表达，
   不要使用'碳水化合物''血糖生成指数'等专业术语：
   糖醋里脊含糖30g，GI值高..."
- 输出："李大爷，这个糖醋里脊裹了很多糖，您有糖尿病不能吃哦"

Step 6: 语音播报（端侧）
- TTS引擎：OPPO自带的中文TTS（支持方言）
- 语速：比正常慢20%，音量自动调高
性能指标：
识别速度：端侧<2秒，云端<5秒
准确率：常见菜品>90%，生僻菜品>75%
用户满意度目标：老人能听懂>95%
功能模块2：冰箱智能管理
使用场景：
王奶奶打开冰箱，看到昨天的剩菜、前天买的鸡蛋，不确定还能不能吃。
操作流程：
1.打开冰箱，点击"拍冰箱"按钮
2.手机自动识别冰箱内的食材
3.语音提醒："那个青菜已经3天了，别吃了容易拉肚子"
4.推荐食谱："鸡蛋还新鲜，可以做个番茄鸡蛋汤"
技术实现：
物体检测：YOLOv8-Nano识别常见食材（蔬菜、肉类、蛋奶）
日期推理： 
o用户首次拍摄时记录时间戳
o结合食材保质期数据库（白菜3天、鸡蛋15天）
o计算剩余保质期
食谱生成：调用大模型API 
o输入：现有食材+用户健康限制
o输出：简单易做的家常菜食谱
创新价值：
减少食物浪费（响应环保主题）
预防食物中毒（老人肠胃敏感）
降低买菜频率（方便独居老人）
功能模块3：语音情感陪伴日记
使用场景：
每天晚饭后，赵大爷习惯和APP"聊聊天"，说说今天吃了什么。
操作流程：
1.点击"今天吃了啥"按钮
2.语音输入："今天中午吃了红烧肉，晚上喝了粥"

4.每周日生成周报发送给子女微信
技术实现：

关键词提取： 
o食物关键词："红烧肉""粥"
o情感关键词："挺好""还行""唉""又是一个人"
情感分析（简化版，避免过度承诺）： 
o不做复杂的情绪识别
o仅检测"孤独信号词"："一个人""没人""冷清"
o触发阈值：连续3天出现孤独词汇 → 提醒子女
周报内容示例：
【爸爸本周饮食报告】

⚠️  营养提醒：红烧肉吃了3次，建议减少油腻
💬 情感观察：爸爸说了2次"一个人吃"，可能有些孤单，
             有空多陪陪他哦

2.3 关键技术攻关点
技术难点1：低质量图片的鲁棒识别
挑战：老年人拍照常见问题
手抖导致模糊
光线暗（餐厅环境）
反光（菜单塑封）
角度歪斜
解决方案：
1.图像增强预处理 
o去模糊：Wiener滤波
o去反光：偏振光补偿算法
o矫正：透视变换
2.多帧融合 
o引导用户连拍3张
o自动选择最清晰的一张
3.降低识别门槛 
o置信度<80%时提示"可以再拍一张吗？"
o提供手动输入兜底方案

2.5 端云协同策略
调度逻辑：
fun processMenuImage(image: Bitmap): Result {
    // Step 1: 端侧OCR（必选）
    val text = paddleOCR.recognize(image)
    
    // Step 2: 查询本地知识库
    val localResult = database.queryDish(text)
    
    if (localResult.confidence > 0.85) {
        // 高置信度，直接使用本地结果
        return localResult
    } else if (networkAvailable && userConsent) {
        // Step 3: 云端增强（需用户授权）
        val cloudResult = cloudAPI.analyzeImage(image, text)
        return cloudResult
    } else {
        // 降级方案：提示用户手动输入
        return showManualInputDialog()
    }
}

# 参赛项目计划书：银发·智膳助手 (ElderCare AI)

**—— 基于端云协同与语义转译的银发族智慧饮食伴侣**

## 一、 项目概述 (Project Overview)

### 1.1 背景与目的

**社会痛点：** 中国60岁及以上人口已超2.6亿，在数字化浪潮中，老年人面临“看不懂新式菜单、记不住慢病禁忌、缺乏情感关怀”的饮食困境。现有的营养App多为年轻人设计，术语晦涩、操作复杂，难以满足银发族的健康管理需求。
**项目目的：** 打造一款专为老年人设计的“零门槛”饮食助手。通过视觉语义转译技术将专业数据转化为“大白话”，结合语音情感特征感知老人状态，消除数字鸿沟，守护饮食安全。

### 1.2 核心创新点 (对齐评分：创新性 40分)

1. **知识图谱驱动的“大白话”翻译引擎：** 突破单纯OCR识别，通过视觉大模型将识别到的菜名（如“地三鲜”）深度转译为生活化表达（“土豆茄子油炸的，油大”），实现营养知识的适老化降维。
2. **伴随式情感感知交互：** 利用端侧音频特征分析（语调、语速、停顿），感知老人在交互时的情绪状态（如孤独、疲惫），动态调整AI播报语速与语调，并生成包含情绪信号的“子女周报”。
3. **杀手级场景：冰箱智能保质期管理：** 针对老人节俭、易误食过期食品的习惯，通过“拍冰箱”识别食材并自动记录存放时间，提供过期预警，预防食物中毒。

---

## 二、 技术方案 (Technical Scheme) (对齐评分：技术深度 25分)

### 2.1 总体架构

采用**“端侧优先、云端增强”**的端云协同架构，确保隐私安全与响应速度。

* **端侧 (OPPO手机)：** 部署轻量化 `PaddleOCR`、`YOLOv11-Nano` 及本地知识库，负责实时文字提取、食材检测及离线TTS播报。
* **云侧 (算力支撑)：** 按需调用视觉大模型 (Vision LLM) 进行生僻菜品分析，利用大语言模型 (LLM) 进行语义转译。

### 2.2 关键技术细节

1. **适老化视觉优化：** 针对老人拍照易“手抖、模糊、反光”的问题，引入端侧图像增强算法（去模糊、透视矫正），提升低质量图像下的识别鲁棒性。
2. **语义转译逻辑 (Prompt Engineering)：** 建立“专业词汇-生活化语言”映射库，通过LLM将GI值、热量、脂肪含量等抽象指标转化为“能不能吃、建议吃多少”的具象建议。
3. **端侧隐私保护：** 所有的面部表情（如可选）与语音特征向量化均在端侧完成，云端仅处理脱敏后的文本标签，符合《个人信息保护法》要求。

## 三、 功能概述 (Functional Description)

1. **拍菜单识健康：** 一键拍照，AI 语音播报菜品成分、健康风险及替代建议。
2. **拍冰箱防变质：** 识别冰箱内存货，自动倒计时保质期，提醒老人优先食用即将过期的蔬菜。
3. **语音日记与情感伴随：** 老人通过语音记录饮食，AI 以温暖语调回应，并分析情绪词汇，捕捉独居老人的心理波动。
4. **子女守护中心：** 子女远程录入体检报告，设置“过敏/禁忌”红线，实现跨时空的健康守护。
